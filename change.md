## 只改提示词就能明显变难：你要加的 4 条硬约束

把下面这些**同时**加到 `operate_calculation` + `final compress`（以及 step2/3）里，效果立竿见影。

### 1) 禁止“解释型选项”（最关键）

加入这条（直接粘进去）：

```text
选项泄露禁令：
- A-D 选项只允许输出“数值/区间/等级标签”（长度≤12字），不得附带原因、现象、分布描述或任何与图像特征对应的解释；
- 禁止出现“高度重合/明显分离/只在外圈/几乎只在背景”等可被直接图像匹配的词。
```

### 2) 强制“可核验”的视觉量化（点映射必须能数出来）

```text
可核验视觉量化：
- 若题材是元素点映射/点云分布，必须使用可计数/可网格估算的量化证据（如：指定ROI内点数比、内外环点数比、四象限计数、网格覆盖格数）；
- 禁止仅凭“重合程度/略富集/看起来更密集”这类主观描述来定义比例或指标。
```

### 3) 禁止编造公式/经验关系（防止“自造可算捷径”）

```text
来源约束：
- 不得新增任何未在 reference/context/steps evidence 中出现的变量定义、经验关系或阈值；
- 若参考信息不足以支撑计算题，必须改写为“读图可核验的计数/比较/排序”型问题，而不是自造公式。
```

### 4) 让数值选项“贴近”而不是一眼排除

```text
同质且近邻：
- 四个数值选项必须同单位、同小数位；
- 最大值/最小值 ≤ 1.25（或差值不超过正确值的±15%），避免跨度过大导致秒选。
```

额外一招（不是提示词，但能从系统层面把“简单题”挡掉）

你们 README 里提到“Text-Only 盲测可选”，但当前看起来没启用。建议加一个硬门槛：

* **Text-only solver 也选对 ⇒ 直接判“文本捷径”，重写/丢弃**
  这能专治“选项描述太明显”的题。
